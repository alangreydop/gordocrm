# Sesión 2026-04-07 — Fix Portal de Clientes

## Problema inicial
El "Área de cliente" en la web estaba roto:
- Mostraba un formulario pidiendo "ID de pedido"
- Llamaba a un endpoint inexistente (`/api/order/:id`)
- El flujo del cliente se frenaba ahí

## Solución implementada

### 1. Rediseño completo del portal (`/Users/alangreydop/gordo/src/pages/portal.astro`)

**Antes:**
- Formulario de ID de pedido
- UI genérica, no on-brand
- Endpoint falso

**Ahora:**
- Formulario login email/password
- Diseño on-brand con estilo Grande&Gordo:
  - Hero section con kicker, título grande, subtítulo
  - Card oscura con gradientes sutiles
  - Inputs estilizados con focus en accent #C4165A
  - Botón con gradiente y shadow
  - Sección alternativa para acceso directo al CRM
- Conecta al endpoint real: `/api/portal/auth/login`
- Redirige al dashboard `/client/` tras login

### 2. Configuración de dominio personalizado

**Problema:** Las URLs apuntaban a `gordocrm.gordoagency.com` (dominio inexistente)

**Solución:**
1. Usuario creó subdominio `crm.grandeandgordo.com` en Cloudflare Workers
2. Añadido CNAME en DNS:
   - Name: `crm`
   - Target: `gordocrm-api-production.alangreydop.workers.dev`
3. Actualizadas todas las URLs en el portal a `crm.grandeandgordo.com`
